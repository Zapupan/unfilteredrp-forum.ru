# Настройка сервера для форума

## Проблема

Сейчас все данные хранятся в `localStorage` браузера каждого пользователя. Это означает, что каждый пользователь видит только свои данные.

## Решение

Создан простой Node.js сервер с SQLite базой данных, который будет хранить все данные на сервере.

## Установка

1. Убедитесь, что у вас установлен Node.js (версия 14 или выше)
   - Проверьте: `node --version`

2. Установите зависимости:
```bash
npm install
```

3. Запустите сервер:
```bash
npm start
```

Сервер запустится на порту 3000 (или на порту, указанном в переменной окружения PORT).

## Что нужно сделать дальше

⚠️ **ВНИМАНИЕ**: Это базовая версия сервера, которая реализует только регистрацию и вход!

Для полной работы форума нужно:
1. Реализовать все остальные API endpoints (посты, комментарии, сообщения и т.д.)
2. Изменить `script.js` чтобы использовать реальный API вместо localStorage
3. Настроить хостинг для Node.js приложения

## Развертывание на хостинге

### Если хостинг поддерживает Node.js:

1. Загрузите все файлы на хостинг
2. Установите зависимости: `npm install`
3. Запустите сервер: `npm start`
4. Настройте процесс-менеджер (PM2) для постоянной работы:
   ```bash
   npm install -g pm2
   pm2 start server.js
   pm2 save
   ```

### Если хостинг НЕ поддерживает Node.js:

Вам нужно:
- Использовать VPS (виртуальный сервер)
- Или использовать Firebase/Supabase вместо своего сервера
- Или переписать на PHP (если хостинг поддерживает PHP)

## Важные замечания

1. **Безопасность**: Текущая реализация использует простой хеш паролей. В продакшене нужно использовать bcrypt!
2. **SQL Injection**: Код уязвим к SQL injection. Нужно использовать параметризованные запросы.
3. **Аутентификация**: Токены сейчас простые строки. Нужно использовать JWT.

## Альтернативные решения

Если настройка сервера сложна, можете использовать:
- **Firebase** - самый простой вариант
- **Supabase** - бесплатная альтернатива Firebase
- **Backendless** - еще один вариант BaaS

Эти сервисы не требуют настройки сервера и работают "из коробки".
